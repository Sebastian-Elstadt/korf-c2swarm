# 2026-03-05

## Nodus

### Summary

Implemented asymmetric security for Nodus as separate lib project to abstract dependencies.

### nodus_asym_sec

Each Nodus instance needs an asymmetric keypair for securing communications. I opted for the `ed25519_dalek` crate, which provides a simple and efficient implementation of the Ed25519 signature scheme. But the crate seems to be a bit outdated and requires older versions of the rand crate. I decided to move the asymmetric security code to its own lib project to isolate usage of the older crates and their dependencies, in case I want to use newer rand crate versions in the main binary crate. And it turned out to be a pretty clean implementation.

The `lib.rs` file exposes a simple trait, `AsymSecHandler`, and implementations for it. Currently there is only the `ed25519_dalek` implementation, but this way if I find the crate - which is already a outdated, dropping my confidence in it - to be inadequate, I can easily swap the implementation.

I am likely to reuse this pattern for other dependencies... However, I am not sure if it is a good idea to expose a `get_implementation` function in the lib project, letting the lib decide what implementation to use. But that's a bridge to cross when I get there.
